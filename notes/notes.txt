AWS — Amazon Web Services
=========================

AWS is Amazon's cloud platform. Instead of buying and maintaining your own servers, you rent exactly what you need — computing power, storage, databases, networking — and pay only for what you use. Everything runs in Amazon's data centers around the world, so your apps can be fast and available globally.


Core Concepts
-------------

Instance
  An EC2 instance is essentially a virtual computer running in the cloud. You can start it, stop it, resize it, or terminate it at any time. There's no physical hardware to worry about — AWS handles all of that.

Instance Type
  This defines how powerful your instance is — how many CPUs it has, how much memory, and what kind of storage. For example, a compute-optimized type is great for heavy processing, while a memory-optimized type handles large in-memory datasets. Choose based on what your workload actually needs.


Images
------

AMI — Amazon Machine Image
  An AMI is a pre-packaged snapshot of an operating system, software, and configuration. When you launch a new instance, you pick an AMI — it's like choosing what comes pre-installed on your machine. AWS offers a catalog of official AMIs (Amazon Linux, Ubuntu, Windows, etc.), and you can also create your own.


Access & Security
-----------------

Key Pairs
  A key pair is how you securely SSH into your EC2 instance. AWS stores the public key on the instance; you download and keep the private key (.pem file). Without the private key, there's no way to log in — so keep it safe and never share it.

IAM — Identity and Access Management
  IAM is how you control who can do what inside your AWS account. You create users and roles, assign them specific permissions (read-only, full access, etc.), and make sure people and services only have access to what they actually need. It's your first line of defense for account security.

Security Groups
  Think of a security group as a virtual firewall around your instance. You write rules that define which traffic is allowed in (inbound) and out (outbound). For example: allow SSH on port 22, allow HTTP on port 80, block everything else. Rules can be scoped to specific IP ranges for tighter control.


Networking
----------

Load Balancer
  A load balancer sits in front of your fleet of instances and distributes incoming requests across them. If one instance is under heavy load, traffic gets routed to others. This keeps your application responsive and prevents any single instance from becoming a bottleneck. It also improves availability — if one instance goes down, traffic automatically shifts to healthy ones.


Storage
-------

Volumes (EBS — Elastic Block Store)
  A volume is a virtual hard drive attached to your EC2 instance. It's where your operating system, application code, and data live. You can resize a volume without stopping the instance, and you can detach it from one instance and attach it to another — like unplugging a hard drive and plugging it into a different computer.

Snapshots
  A snapshot is a point-in-time backup of a volume, stored in S3. If something goes wrong — a bad deployment, accidental deletion, disk corruption — you can restore the volume from a snapshot. It's a good habit to take snapshots before making major changes to a running system.
