S3 — Simple Storage Service
============================

What is S3?
-----------
S3 is AWS's object storage service — designed to store and retrieve any amount of data from anywhere on the internet. You store files (called objects) inside containers (called buckets). Think of a bucket as a top-level folder in the cloud, except it's globally accessible, virtually unlimited in size, and built for high durability (AWS promises 99.999999999% — "eleven nines" — durability).


Key Concepts
------------

Bucket
  A bucket is the top-level container for all your files. Bucket names must be globally unique across every AWS account in the world, so pick something specific. You also choose a region where the bucket physically lives — keeping it close to your users reduces latency.

Object
  An object is any file you store in S3 — images, videos, logs, backups, CSVs, whatever. Each object has:
    - a Key   : its unique name/path inside the bucket (e.g. "uploads/photo.jpg")
    - a Value : the actual file content
    - Metadata: optional key-value pairs (content type, custom tags, etc.)


Storage Classes
---------------
S3 offers different storage tiers so you only pay for what you need:

  Standard              Frequently accessed data. Highest availability, highest cost.
                        Best for active production data.

  Infrequent Access     Data you rarely need but must be able to retrieve quickly.
  (S3-IA)               Lower cost than Standard, small retrieval fee.

  Glacier               Long-term archival storage. Very cheap, but retrieval
                        takes minutes to hours. Great for compliance archives.

  Glacier Deep Archive  The cheapest option. Retrieval can take up to 12 hours.
                        Use for data you almost never need but must legally retain.


Permissions & Access Control
-----------------------------
By default, every bucket and object is private — nothing is publicly accessible unless you explicitly allow it.

  Bucket Policies       JSON-based rules applied at the bucket level. Fine-grained
                        control over who can read, write, or delete objects.

  ACLs                  An older per-object or per-bucket access control mechanism.
                        Bucket policies are generally preferred for new setups.

  Block Public Access   A master safety switch. Even if a bucket policy accidentally
                        allows public access, this setting overrides it. Keep it on
                        unless you're intentionally hosting public content.


Useful Features
---------------

Versioning
  Enable versioning on a bucket and S3 keeps every version of every file. Accidentally
  overwrote an important file? Deleted something you shouldn't have? You can restore
  any previous version instantly. Highly recommended for critical data.

Static Website Hosting
  You can serve a static website — HTML, CSS, JavaScript, images — directly from an
  S3 bucket with zero server setup. Just upload your files, enable the feature, and
  point your domain at it.

Lifecycle Rules
  Automate storage cost management by defining rules like: "Move objects to Glacier
  after 30 days of no access" or "Delete objects older than 1 year." Set it once and
  S3 handles it automatically.


Pricing
-------
S3 charges you for three things:

  - Storage         How many GB you're storing, per month.
  - Requests        Every API call costs a tiny amount (GET, PUT, DELETE, LIST, etc.).
  - Data Transfer   Moving data out of AWS to the internet costs money.
                    Transfers between AWS services in the same region are free.
